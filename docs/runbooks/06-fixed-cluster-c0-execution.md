# Fixed-Cluster C0 Execution

## Purpose

This runbook describes how to regenerate the complete `C0` fixed-cluster execution track.

`C0` is the historical fixed-cluster characterization track of the LocalAI worker-mode benchmark suite. Unlike provider-backed cycles, it does not create, resize, or destroy infrastructure through `proxmox-k3s`. It assumes that an external Kubernetes or K3s cluster is already reachable through a fixed kubeconfig file and then executes the LocalAI worker-mode characterization workflow against that cluster.

The objective of this runbook is to regenerate the full `C0` evidence set, including:

- fixed-cluster Kubernetes validation;
- namespace and foundation deployment;
- LocalAI worker-mode baseline deployment;
- standalone precheck and API validation;
- official baseline executions;
- exploratory load validation;
- individual pilot sweeps;
- consolidated pilot campaign;
- technical diagnosis;
- reporting;
- completion-gate evaluation;
- freeze snapshot.

`C0` must be kept separate from the provider-backed cycles `C1` through `C9`. It is a fixed-cluster execution path and must not be interpreted as a dynamically provisioned infrastructure cycle.

## C0 Execution Model

`C0` follows this execution model:

```text
fixed cluster access
→ namespace and foundation resources
→ LocalAI worker-mode deployment
→ standalone validation
→ official baseline
→ exploratory validation
→ individual pilot sweeps
→ consolidated pilot campaign
→ technical diagnosis
→ reporting
→ completion gate
→ freeze
```

The fixed cluster is not provisioned by this benchmark suite. The benchmark suite only consumes a kubeconfig file, applies Kubernetes resources, executes workload generation, and writes reproducible artifacts under `results/experimental-cycles/C0/`.

## Execution Safety Rules

Before running C0, observe the following rules.

1. The kubeconfig must point to the intended fixed cluster.
2. The namespace `localai-benchmark` must be considered absent at the beginning of a clean regeneration.
3. The C0 workflow must not call `proxmox-k3s` provisioning commands.
4. Results generated by C0 must remain under the C0 artifact tree whenever possible.
5. Unsupported scenarios, such as scenarios that cannot be scheduled under fixed-cluster constraints, must be preserved as valid evidence rather than manually rewritten as successful measurements.
6. Do not mix C0 artifacts with provider-backed artifacts from `C1` through `C9`.

## Step 15 — Run Individual Pilot Sweeps

The consolidated pilot campaign is the primary aggregate execution path, but the individual pilot sweep launchers are useful when regenerating the full fixed-cluster evidence set. Run them before or after the consolidated campaign, depending on the desired artifact layout.

Some scenarios may be unsupported on the fixed cluster. Continue collecting evidence for the remaining scenarios and inspect unsupported manifests after execution.

### Windows PowerShell

```powershell
$Replicas = @("A", "B", "C")

$PilotFamilies = @(
  @{ Name = "worker-count"; Script = ".\scripts\load\pilot\Start-PilotWorkerCountSweep.ps1"; Scenarios = @("W1", "W2", "W3", "W4") },
  @{ Name = "workload"; Script = ".\scripts\load\pilot\Start-PilotWorkloadSweep.ps1"; Scenarios = @("WL1", "WL2", "WL3") },
  @{ Name = "models"; Script = ".\scripts\load\pilot\Start-PilotModelSweep.ps1"; Scenarios = @("M1", "M2", "M3", "M4") },
  @{ Name = "placement"; Script = ".\scripts\load\pilot\Start-PilotPlacementSweep.ps1"; Scenarios = @("PL1", "PL2") }
)

foreach ($Family in $PilotFamilies) {
  foreach ($Scenario in $Family.Scenarios) {
    foreach ($Replica in $Replicas) {
      powershell.exe -NoProfile -ExecutionPolicy Bypass -File $Family.Script `
        -Scenario $Scenario `
        -Replica $Replica `
        -Kubeconfig $Kubeconfig `
        -Namespace $Namespace `
        -BaseUrl $BaseUrl `
        -PrecheckConfig $PrecheckProfile `
        -PhaseConfig $PhaseProfile `
        -ProtocolConfig $ProtocolProfile `
        -ClusterCaptureConfig $ClusterCaptureProfile `
        -MetricSetConfig $MetricSetProfile `
        -AutoApplyK8s

      if ($LASTEXITCODE -ne 0) {
        Write-Warning "Pilot family $($Family.Name), scenario $Scenario, replica $Replica completed with a non-zero exit code. Continue and inspect the corresponding artifacts."
      }
    }
  }
}
```
